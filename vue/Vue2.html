# 全局API

名称描述
Vue.extend( options )在新窗口打开使用基础 Vue 构造器，创建一个“子类”。参数是一个包含组件选项的对象。data 选项是特例，需要注意 - 在 Vue.extend() 中它必须是函数
[Vue.nextTick( callback, context] )在新窗口打开在下次 DOM 更新循环结束之后执行延迟回调。在修改数据之后立即使用这个方法，获取更新后的 DOM。
Vue.set( target, propertyName/index, value )在新窗口打开向响应式对象中添加一个 property，并确保这个新 property 同样是响应式的，且触发视图更新。它必须用于向响应式对象上添加新 property，因为 Vue 无法探测普通的新增 property (比如 this.myObject.newProperty = 'hi')
Vue.delete( target, propertyName/index )在新窗口打开删除对象的 property。如果对象是响应式的，确保删除能触发更新视图。这个方法主要用于避开 Vue 不能检测到 property 被删除的限制，但是你应该很少会使用它。
[Vue.directive( id, definition] )在新窗口打开注册或获取全局指令。
[Vue.filter( id, definition] )在新窗口打开注册或获取全局过滤器。
[Vue.component( id, definition] )在新窗口打开注册或获取全局组件。注册还会自动使用给定的 id 设置组件的名称
Vue.use( plugin )在新窗口打开安装 Vue.js 插件。如果插件是一个对象，必须提供 install 方法。如果插件是一个函数，它会被作为 install 方法。install 方法调用时，会将 Vue 作为参数传入。该方法需要在调用 new Vue() 之前被调用。
Vue.mixin( mixin )在新窗口打开全局注册一个混入，影响注册之后所有创建的每个 Vue 实例。插件作者可以使用混入，向组件注入自定义的行为。不推荐在应用代码中使用。
Vue.compile( template )在新窗口打开将一个模板字符串编译成 render 函数。只在完整版时可用。
Vue.observable( object )在新窗口打开让一个对象可响应。Vue 内部会用它来处理 data 函数返回的对象。
Vue.version在新窗口打开提供字符串形式的 Vue 安装版本号。这对社区的插件和组件来说非常有用，你可以根据不同的版本号采取不同的策略。

# 数据选项

名称类型描述
data在新窗口打开Object|Function,组件的定义只接受 function。Vue 实例的数据对象。实例创建之后，可以通过 vm.$data 访问原始数据对象。
props在新窗口打开Array<string>Objectprops 可以是数组或对象，用于接收来自父组件的数据。
propsData在新窗口打开{ [key: string]: any }只用于 new 创建的实例中。创建实例时传递 props。主要作用是方便测试。
computed在新窗口打开{ [key: string]: Function|{ get: Function, set: Function } }计算属性。
methods在新窗口打开{ [key: string]: Function }方法
watch在新窗口打开{ [key: string]: string|Function|Object|Array }监听属性变化

# DOM选项

名称类型描述
el在新窗口打开string|Element提供一个在页面上已存在的 DOM 元素作为 Vue 实例的挂载目标。可以是 CSS 选择器，也可以是一个 HTMLElement 实例。
template在新窗口打开string一个字符串模板作为 Vue 实例的标识使用。模板将会替换挂载的元素。挂载元素的内容都将被忽略，除非模板的内容有分发插槽。
render在新窗口打开(createElement: () => VNode) => VNode字符串模板的代替方案，允许你发挥 JavaScript 最大的编程能力。该渲染函数接收一个 createElement 方法作为第一个参数用来创建 VNode。
renderError在新窗口打开(createElement: () => VNode, error: Error) => VNode当 render 函数遭遇错误时，提供另外一种渲染输出。其错误将会作为第二个参数传递到 renderError。这个功能配合 hot-reload 非常实用。

# 生命周期选项

名称描述
beforeCreate在新窗口打开实例创建前：数据和模板均未获取到
created在新窗口打开实例创建后: 最早可访问到 data 数据，但模板未获取到
beforeMount在新窗口打开数据挂载前：模板已获取到，但是数据未挂载到模板上。
mounted在新窗口打开数据挂载后： 数据已挂载到模板中
beforeUpdate在新窗口打开模板更新前：data 改变后，更新数据模板前调用
updated在新窗口打开模板更新后：将 data 渲染到数据模板中
activated在新窗口打开被 keep-alive 缓存的组件激活时调用。
deactivated在新窗口打开被 keep-alive 缓存的组件停用时调用。
beforeDestroy在新窗口打开实例销毁之前调用。在这一步，实例仍然完全可用。
destroyed在新窗口打开实例销毁后调用。该钩子被调用后，对应 Vue 实例的所有指令都被解绑，所有的事件监听器被移除，所有的子实例也都被销毁。
errorCaptured在新窗口打开当捕获一个来自子孙组件的错误时被调用。此钩子会收到三个参数：错误对象、发生错误的组件实例以及一个包含错误来源信息的字符串。此钩子可以返回 false 以阻止该错误继续向上传播。

# 组合选项

名称类型描述
parent在新窗口打开Vue instance指定已创建的实例之父实例，在两者之间建立父子关系。子实例可以用 this.$parent 访问父实例，子实例被推入父实例的 $children 数组中。
mixins在新窗口打开Array<Object>mixins 选项接收一个混入对象的数组。这些混入对象可以像正常的实例对象一样包含实例选项，这些选项将会被合并到最终的选项中，使用的是和 Vue.extend() 一样的选项合并逻辑。也就是说，如果你的混入包含一个 created 钩子，而创建组件本身也有一个，那么两个函数都会被调用。Mixin 钩子按照传入顺序依次调用，并在调用组件自身的钩子之前被调用。
extends在新窗口打开Object|Function允许声明扩展另一个组件 (可以是一个简单的选项对象或构造函数)，而无需使用 Vue.extend。这主要是为了便于扩展单文件组件。这和 mixins 类似。

# 其他选项

名称类型描述
name在新窗口打开string允许组件模板递归地调用自身。注意，组件在全局用 Vue.component() 注册时，全局 ID 自动作为组件的 name。
delimiters在新窗口打开Array<string>改变纯文本插入分隔符。
functional在新窗口打开boolean使组件无状态 (没有 data) 和无实例 (没有 this 上下文)。他们用一个简单的 render 函数返回虚拟节点使它们渲染的代价更小。
model在新窗口打开{ prop?: string, event?: string }允许一个自定义组件在使用 v-model 时定制 prop 和 event。默认情况下，一个组件上的 v-model 会把 value 用作 prop 且把 input 用作 event，但是一些输入类型比如单选框和复选框按钮可能想使用 value prop 来达到不同的目的。使用 model 选项可以回避这些情况产生的冲突。
inheritAttrs在新窗口打开boolean默认情况下父作用域的不被认作 props 的 attribute 绑定 (attribute bindings) 将会“回退”且作为普通的 HTML attribute 应用在子组件的根元素上。
comments在新窗口打开boolean当设为 true 时，将会保留且渲染模板中的 HTML 注释。默认行为是舍弃它们。

# 自定义组件的v-model

<template>
  <input type="checkbox" :checked="checked" @change="updateValue" />
</template>
<script>
export default {
  props: {
    checked: Boolean,
  },
  model: {
    prop: "checked", //绑定的属性名
    event: "change", //绑定的事件
  },
  methods: {
    updateInput(e) {
      this.$emit("change", e.target.checked);
    },
  },
};
</script>

const CompressionWebpackPlugin = require('compression-webpack-plugin');
const productionGzipExtensions = /\.(js|css|json|txt|html|ico|svg)(\?.*)?$/i;

module.exports = {
    configureWebpack: config => {
        if (IS_PROD) {
            const plugins = [];
            plugins.push(
                new CompressionWebpackPlugin({
                    filename: '[path].gz[query]',

还可以开启比gzip体验更好的Zopfli压缩详见https://webpack.js.org/plugins/compression-webpack-plugin在新窗口打开

npm i --save-dev @gfx/zopfli brotli-webpack-plugin

const CompressionWebpackPlugin = require('compression-webpack-plugin');
const zopfli = require("@gfx/zopfli");
const BrotliPlugin = require("brotli-webpack-plugin");
const productionGzipExtensions = /\.(js|css|json|txt|html|ico|svg)(\?.*)?$/i;

module.exports = {
    configureWebpack: config => {
        if (IS_PROD) {
            const plugins = [];
            plugins.push(
                new CompressionWebpackPlugin({
                    algorithm(input, compressionOptions, callback) {
                      return zopfli.gzip(input, compressionOptions, callback);
                    },
                    compressionOptions: {
                      numiterations: 15
                    },
                    minRatio: 0.99,
                    test: productionGzipExtensions
                })
            );
            plugins.push(
                new BrotliPlugin({
                    test: productionGzipExtensions,
                    minRatio: 0.99
                })
            );
            config.plugins = [
                ...config.plugins,
                ...plugins
            ];
        }
    }
}
